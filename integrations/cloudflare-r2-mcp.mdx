Cloudflare R2 Remote MCP Worker gives remote MCP-based hosts, such as ChatGPT, a personal bridge to files mirrored on Cloudflare R2, so they can work with your storage without needing a local MCP server or a native R2 connector. The Worker deploys to Cloudflare's edge, stays available without a machine running in the background, and lets ChatGPT read from and write to your R2 bucket directly through an MCP endpoint.

## How It Works

The bridge connects a remote MCP host to an R2 bucket through a deployed Cloudflare Worker. The Worker registers as an MCP server, exposes R2 storage operations as MCP tools, and handles authentication so that only authorized users can reach the endpoint. There is no local server to install and no machine to keep running.

<Tip>
Use R2 as a persistence layer for material that needs to outlast a single AI session — project context, reviewed links, working drafts, signal libraries, and session handoff notes. Write once, read across any session or AI host that connects to the Worker.
</Tip>

## Why R2 + MCP

R2 combined with a remote MCP Worker solves a specific problem that comes up in AI-assisted workflows: AI sessions are stateless, but your work is not.

Files, drafts, project context, session notes, and data artifacts don't need to live on your local machine to be useful across multiple AI conversations. R2 gives you a durable object store. The MCP Worker gives ChatGPT a way to read from and write to that store without you manually uploading, downloading, and pasting content between sessions.

Key properties that make this combination practical:

- **No local server required.** The Worker runs on Cloudflare's edge. You deploy it once and it stays available without a machine running in the background.
- **Works with ChatGPT and other MCP hosts.** Any MCP-compatible client that supports custom connectors can connect to the Worker endpoint.
- **Persistent across sessions.** Files written to R2 in one ChatGPT session are readable in the next one. Context, research signals, drafts, and data can accumulate across time.
- **Scoped to one bucket.** The Worker binds to one R2 bucket per deployment, keeping the default access surface narrow.
