DeadPunk OS is a browser-based fake operating system that serves as the public face of the XxYouDeaDPunKxX project hub. It lives at the GitHub Pages URL for MaRCoRe's profile repository, and it is the first thing you encounter when you show up at the hub looking for something real. Is it a practical tool? No. Is it broken? Also no. It's avant-garde. There is a difference, and DeadPunk OS would like you to appreciate it.

## What It Is

DeadPunk OS is not a real operating system. It does not run processes, manage memory, or have a kernel. What it is, is a browser-based interactive experience styled to look and feel like a desktop OS — boot sequence, terminal aesthetic, fake modules, questionable system status messages, and all.

It was built as a GitHub Pages deployment for the `XxYouDeaDPunKxX` profile repository. Its job is to be the weird, honest front door to a collection of projects that don't fit neatly into a portfolio template. The README puts it plainly: these projects come from real use, not from trying to look impressive. DeadPunk OS reflects that.

If you expected a polished landing page, the OS has noted your expectation and is choosing to ignore it.

## The Aesthetic

DeadPunk OS is the public expression of a specific design stance: build for real use, not for appearances. The projects in the MaRCoRe hub are practical AI tools, behavioral protocols, and workflow systems built to solve actual problems. They are not portfolio pieces.

DeadPunk OS applies that same stance to the hub itself. Instead of clean cards, gradient backgrounds, and confidence-inspiring copy about scalable solutions, you get a fake OS with questionable system status and duct tape logic doing its best.

The aesthetic is technical, deliberately rough, and indifferent to what looks impressive. Everything is fine.

Build — pure spite & prompts.  
Stack — held by duct tape under threat.  
Mood — weird ideas, real problems.

That is the vibe. The projects are serious. The landing page is not trying to sell you anything.
