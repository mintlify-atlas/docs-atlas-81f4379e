ChatGPT Local Agent MCP is a local MCP (Model Context Protocol) server that bridges ChatGPT to your Windows desktop environment. Instead of copying error logs, file snippets, and terminal output back and forth into a chat window, you give ChatGPT supervised access to the actual machine where the work lives — so it can read your files, run commands, inspect Git state, open browser sessions, and take screenshots while you stay in control at every step.

## What It Enables

ChatGPT stays the brain. This gives it hands on your computer — scoped, supervised, and intentionally explicit about what it can reach.

## The Control Model

The key design decision in ChatGPT Local Agent MCP is that you remain in control at all times. The server does not run autonomously. ChatGPT can only reach what you configure, and only through the scopes you explicitly enable.

Treat ChatGPT's access like you would treat a human assistant sitting at your keyboard. The server runs with the permissions of the Windows account that starts it. If that account can read a file or launch a command, the exposed tools can reach the same surface. Limit access to project folders and enable only the capabilities you actually need.

<Note>
ChatGPT Local Agent MCP is **Windows-only**. The installer, runtime paths, workspace profile defaults, and desktop automation tools are built for Windows. It is not designed to run on macOS or Linux.
</Note>

## Safety Considerations

<Warning>
**This project is intentionally capable, not sandboxed.**

Before connecting ChatGPT, understand the full tool surface you are enabling. File access, command execution, browser automation, and desktop automation all run with the permissions of the Windows account running the server. Limit scope to the project folders and capabilities you actually need. Never expose the server publicly without proper authentication in place.

ChatGPT also has its own safety layer. Some write, submit, and browser actions may be blocked on the OpenAI side regardless of what the MCP server exposes.
</Warning>
